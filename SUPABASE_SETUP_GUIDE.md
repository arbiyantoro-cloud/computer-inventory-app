# Setup Supabase - Panduan Lengkap\n\n## 📋 Daftar Checklist\n- [ ] Langkah 1: Buat Project Supabase\n- [ ] Langkah 2: Setup Database Schema\n- [ ] Langkah 3: Enable Authentication\n- [ ] Langkah 4: Get API Credentials\n- [ ] Langkah 5: Configure .env.local\n- [ ] Langkah 6: Create Demo Users\n- [ ] Langkah 7: Verify Setup\n\n---\n\n## 🔧 LANGKAH 1: Buat Project Supabase\n\n### 1a. Buka Supabase\n1. Kunjungi: https://supabase.com\n2. Klik **\"Sign Up\"** atau **\"Sign In\"** jika sudah punya akun\n3. Pilih metode (GitHub, Google, atau email)\n\n### 1b. Buat Project Baru\n1. Dashboard → Klik tombol **\"+ New Project\"**\n2. Atau jika di halaman organizasi → **\"New project\"**\n\n### 1c. Isi Konfigurasi Project\n\n**Field-field yang perlu diisi:**\n\n```\nProject Name:        computer-inventory-app\nDatabase Password:   (buat password yang kuat, minimal 12 karakter)\n                     Contoh: P@ssw0rd!2024Secure\nRegion:              Asia Pacific (Singapore) - ap-southeast-1\nPricing Plan:        Free (sufficient untuk testing)\n```\n\n**Contoh Form:**\n```\n┌─ Project Details ───────────────────┐\n│ Project Name: computer-inventory-app │\n│ Database Password: ••••••••••••••   │\n│ Region: ap-southeast-1 (Singapore)  │\n│ Organization: (pilih org Anda)      │\n└─────────────────────────────────────┘\n```\n\n### 1d. Tunggu Project Selesai Dibuat\n- Proses: 5-10 menit\n- Status: \"Provisioning your database...\" → \"Your project is ready\"\n- Dashboard akan tampil setelah selesai\n\n---\n\n## 🗄️ LANGKAH 2: Setup Database Schema\n\n### 2a. Buka SQL Editor\n1. Di Supabase Dashboard → Sidebar kiri → **\"SQL Editor\"**\n2. Atau klik link: https://app.supabase.com/project/[PROJECT_ID]/sql/new\n\n### 2b. Buat Query Baru\n1. Klik **\"+ New Query\"**\n2. Atau klik **\"New SQL query\"**\n\n### 2c. Copy Database Schema\n\n**PENTING: Copy SELURUH isi file `sql/schema.sql` dari repository**\n\n#### Cara Copy:\n```\n1. Buka file: sql/schema.sql di repository\n2. Select ALL (Ctrl+A / Cmd+A)\n3. Copy (Ctrl+C / Cmd+C)\n4. Kembali ke Supabase SQL Editor\n5. Paste ke query box (Ctrl+V / Cmd+V)\n```\n\n### 2d. Execute Query\n1. Klik tombol **\"Run\"** (atau Ctrl+Enter)\n2. Tunggu eksekusi selesai\n3. Lihat output: Harus ada \"✓\" atau \"Success\"\n\n**Expected Output:**\n```\nExecution completed successfully\nQuery processed\nNo errors\n```\n\n**Jika error, pastikan:**\n- ✓ Copy seluruh konten schema.sql\n- ✓ Tidak ada duplikasi\n- ✓ Database kosong (baru)\n- ✓ Tidak ada karakter special yang tidak diizinkan\n\n### 2e. Verifikasi Tables\n1. Sidebar → **\"Table Editor\"**\n2. Lihat daftar tables di sidebar kanan:\n   - [ ] users\n   - [ ] master_barang\n   - [ ] inventaris_komputer\n   - [ ] jadwal_maintenance\n   - [ ] history_maintenance\n   - [ ] target_perawatan\n   - [ ] monitoring_dashboard\n\n**Jika semua sudah muncul → ✓ Schema OK!**\n\n---\n\n## 🔐 LANGKAH 3: Enable Authentication\n\n### 3a. Buka Authentication Settings\n1. Sidebar → **\"Authentication\"**\n2. Submenu → **\"Providers\"**\n\n### 3b. Enable Email Provider\n1. Cari **\"Email\"** di daftar providers\n2. Pastikan status: **\"Enabled\"** (toggle ON)\n3. Jika belum, klik untuk enable\n\n### 3c. Configure Email Settings (Optional)\n1. Sidebar → **\"Authentication\"**\n2. Submenu → **\"Email Templates\"**\n3. Review templates (default sudah OK)\n4. Bisa customize jika ingin\n\n### 3d. Enable Email/Password\n1. Sidebar → **\"Authentication\"**\n2. Submenu → **\"Settings\"**\n3. Section \"User signups\"\n   - [x] Enable email signups (harus enabled)\n   - [ ] Enable phone signups (optional)\n4. Scroll down → **\"Update\"**\n\n---\n\n## 🔑 LANGKAH 4: Get API Credentials\n\n### 4a. Buka API Settings\n1. Sidebar → **\"Settings\"**\n2. Submenu → **\"API\"**\n\n### 4b. Copy Project URL\n1. Cari section **\"Project API Keys\"**\n2. Cari field **\"Project URL\"** atau **\"API URL\"**\n3. Contoh: `https://xxxxxxxxxxxx.supabase.co`\n4. **COPY** ke somewhere aman\n   ```\n   Simpan dengan label: VITE_SUPABASE_URL\n   ```\n\n### 4c. Copy Anon Public Key\n1. Di bawah Project URL, cari **\"Anon Public Key\"**\n2. Tombol **\"Copy\"** atau select & copy\n3. Key panjang dimulai dengan `eyJ...`\n4. **COPY** ke somewhere aman\n   ```\n   Simpan dengan label: VITE_SUPABASE_ANON_KEY\n   ```\n\n### 4d. Save Credentials\nBuatFile teks temporary, misal `credentials.txt`:\n```\nVITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co\nVITE_SUPABASE_ANON_KEY=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...\n```\n\n**⚠️ JANGAN share ke orang lain atau commit ke Git!**\n\n---\n\n## 👤 LANGKAH 5: Create Demo Users\n\n### 5a.
